CONTEXTO E PREMISSAS:
Esse é um proejto desenvolvido ao decorrer de um dos cursos da plataforma Alura, com ideal educacional e de desenvolvimento técnico/profissional.
A ideia é desenvolver uma aplicação web prática, chamada Style Sync, cujo objetivo é que, em uma resposta rápida à demanda, entregue-se um produto profissional que faça login, cadastro de usuários, catálogo de produtos e painél de vendas.

ESCOLHA DE FRAMEWORK:
Considerando os requisitos de projeto, optou-se pelo framework Flask, que alinha boa intergação API-web, baixa curva de aprendizado e resultados imediatos.
Instalação: pip install Flask

DESENVOLVIMENTO DA ARQUITETURA:
Pasta app - contém todo o código fonte;
  - Pasta routes - responsável por gerenciar as rotas da aplicação
    - main.py -   responsável por organizar todo o grupo de rotas relacionais (Blueprint) e transforma todos os dicionários python em json (jsonify)
                  rota principal: boas vindas da plicação, já vem no método padrão GET;
                  rota products: catálogo de produtos da aplicação, já vem no método padrão GET;
                  rota login: entrada de usuários, metódo definido na função como POST
  - __init__.py - cria o app Flask e registra os blueprints do main;
  
Pasta testes - eventuais testes no decorrer do desenvolvimento;
run.py -   responsável pela execução em si da aplicação; 
           modo de execução: debug=True (recarrega o servidor automáticamente quando houver alterações e permite a detecção de erros)

REQUISITOS FUNCIONAIS:
1. Autenticação do usuário para obter um token (POST);
2. Listagem de todos os produtos (GET);
3. Criação de um novo produto (POST);
4. Visualização dos detalhes de um único e existente produto (GET);
5. Atualização de um único e existente produto (PUT);
6. Deleção de um único e existente produto (DELETE);
7. Importação de vendas através de um arquivo (POST);

PYDANTIC:
- O que é?
  É a biblioteca responsável por redigir o "contrato" que nós faremos com nossos dados. Com ela, se os dados não estiverem no padrão definido,
  obteremos um erro.
- Instalação: 
  pip install pydantic
- Impacto na arquitetura:
  Criação da pasta "models" dentro do diretório base com os padrões de dados a serem consumidos pelo sistema.

NOVO REQUISITO DE PROJETO:
A equipe da "StyleSync" adorou nosso progresso e já tem uma nova ideia: eles querem organizar os produtos em Categorias. 
Isso ajudará os clientes a navegarem melhor pelo e-commerce no futuro.

Tarefas:
- Definir os requisitos de dados (o model Categoria);
- Definir os requisitos funcionais (rotas).

Requisitos de dados:
- título: string. Define qual o título exibido qundo a categoria for requisitada;
- id: int;
- subcategorias: opcional e list. Exemplo: a categoria alimentos pode ser dividida em fitness, carnes, grãos...

Requisitos funcionais:
- Criar categoria (POST);
- Exibir todas as categorias disponíveis por ordem alfabética (GET);
- Exibir subcategorias de uma única e existente categoria pelo id dela (GET);
- Editar categoria uma única e existente categoria pelo seu id (PUT).

MONGODB
Agora, para genrenciar o banco de dados da aplicação, será utilizado o MongoDB.
Suas vantagens são flexibilidade, escalabilidade e acesso rápido.

Requisitos:
- MongoDB
- MongoDB Compass

Instalação no python:
- pip install pymongo python-dotenv (Requer que um arquivo .env seja criado na raiz da aplicação para conectar ao banco de dados)

Impacto na arquitetura:
- .env na raiz
- config.py na pasta app, responsável pelas configurações do banco de dados
